# Review Process for Computational Economic Notes

## For Authors

### How to Submit a Paper
1. Fork this repository
2. Create a new folder: `papers/YEAR/paper-XXX/`
3. Add your paper as `paper.md`
4. Include any code, data, or figures
5. Submit a Pull Request with:
   - Title: "Submission: [Your Paper Title]"
   - Description: Brief abstract and key contributions
   - Label: "submission"

### Revision Process
1. Address reviewer comments
2. Push changes to your branch
3. Mark each comment as resolved when addressed
4. Request re-review when ready

## For Reviewers

### Getting Started
1. You'll be assigned via GitHub assignment
2. Find the paper in the "Papers" section
3. Use the review template

### How to Review
1. **Line Comments**: Click on any line in the PR to comment
2. **Suggest Changes**: Use GitHub's suggestion feature
3. **Overall Review**: Use the review template

### Review Criteria
- **Significance**: Contribution to computational economics
- **Originality**: Novel methods or applications
- **Reproducibility**: Code quality and documentation
- **Clarity**: Writing and organization

### Submitting Your Review
1. Navigate to the Pull Request
2. Click "Review changes" in the top right
3. Select "Request changes" or "Approve"
4. Add your summary comments
5. Submit review

## Review Timeline
- **Initial assignment**: Within 3 days of submission
- **Review period**: 14 days
- **Author revision**: 14 days
- **Final decision**: 7 days after revision

## Contact
For questions about the review process, open an issue or contact the editors.
