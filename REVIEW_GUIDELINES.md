# Review Guidelines for Computational Economic Notes

## Review Process
1. **Submission**: Author submits paper via Pull Request
2. **Screening**: Editor screens for basic requirements
3. **Assignment**: 2-3 reviewers assigned
4. **Review**: Reviewers provide feedback using GitHub's review system
5. **Decision**: Editor makes final decision
6. **Revision**: Author revises based on feedback
7. **Publication**: Paper is published

## Review Criteria

### 1. Originality & Contribution
- Does the paper present novel research?
- What is the contribution to computational economics?
- Is the literature properly reviewed?

### 2. Methodology
- Are computational methods appropriate?
- Is the code efficient and well-documented?
- Are statistical methods correctly applied?

### 3. Reproducibility
- Can results be reproduced with provided code?
- Is data properly documented?
- Are dependencies clearly specified?

### 4. Clarity & Organization
- Is the paper well-structured?
- Are explanations clear?
- Are figures and tables informative?

### 5. Technical Quality
- Code quality and documentation
- Mathematical accuracy
- Statistical validity

## How to Review Using GitHub

### Line-by-line Comments
1. Navigate to the "Files changed" tab
2. Click on any line number to add a comment
3. Start a review to group comments
4. Use "Suggest changes" for specific edits

### Review Submission
1. Fill out the review template
2. Choose your decision:
   - **Approve**: Paper is ready
   - **Comment**: General feedback
   - **Request changes**: Needs revision
3. Submit your review

### Timeline
- Initial review: 2 weeks
- Revisions: 1-2 weeks
- Final decision: Within 1 week of resubmission
